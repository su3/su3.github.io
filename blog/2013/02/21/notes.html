Class Method 和 Instance Method 的区别：

  • Class Method 以 + 号开头，它向Class名称呼叫， 例如 [NSDate date]；

  • Instance Method 以 - 号开头，它向 Class 生成的实例呼叫， 例如：[string uppercaseString];

指针和变量

  • 当声明一个整数时 int i = 100; 这个数值是直接存储在内存中的。
  • 当声明一个 object 时，NSString *str = @"Hello"; str 并不直接保存 Hello 这个值，而是记录 Hello 在内存中的地址信息。
  • 存储器确切地知道一个数值要用多大的空间来装，而一个对象却可大可小，因此用更灵活的方式来表示，那就是通过指针来引用它。
  • 所以 int 变量 i 不用 * 号，它就是值本身，而 str 用 * 号，它是指向对象的指针，不是值。

Code Snippet Library

  • Xcode 右侧 Utilities 视窗下，点击 {} 图标，显示 Code Snippet Library 窗口。
  • 这是预置的一些常用函数，可以直接拖放到编辑窗口，短的方法如 if…else…，复杂的例如Core Data Fetch with * Sorting，不需要从空白开始手动写10多行方法。
  • 也可以把自己写的常用方法选取后拖放到 Snippet 中，右键改名。

调整代码缩进

选择代码，右键，structure > re-indent。或者按 Command + i

抽取部分代码生成新的Function

选择代码，右键，Refactor > Extract…，输入函数名称。此操作完成了手工操作的以下3个步骤：

  1. 把这段代码剪切出来；
  2. 命名一个新方法，把代码贴进去；
  3. 在之前剪切出来的位置呼叫它。

Preprocessor directives

#if DEBUG
    NSLog(@"The maximum value of an int is %i", INT_MAX);
#endif

这段代码只在debug时输出，release时不会有。

BOOL值

  • BOOL 用 YES 和 NO，bool 用 true 和 false 表示真假。
  • 可以用大写的 BOOL，也可以用小写的bool，不过建议用大写，因为作为C语言的衍生语言， Objective-C 加入 BOOL 值时，C 语言中还没有 bool 值，因此 Objective-C 的框架中有大量的函数都是用 BOOL 表示布尔值， Objective-C 对大小写是敏感的，为了安全起见，用大写 BOOL 更可靠。
  • 输出 BOOL 值用 %i，例如 NSLog(@"The BOOL value is %i", YES)，返回结果用1和0指代真假。

Float值

  • 永远在 float 值后面加 f，例如 2.1f，因为可能会当 double 看待，影响精度和效率。

Loops

  • break 跳出循环；
  • continue 不往下执行了，回到起始位置

操作符和表达式

int highScore = (playerOne > playerTwo) ? playerOne : playerTwo;

括号中为条件，当条件为真则 highScore 的值等于 playerOne，否则为 playerTwo。

typedef

声明参数，不赋值则默认第一个参数为0，依次排序赋值。这样就不需要去记哪个数字代表的具体含义了。

typedef enum {window = 99, aisle = 199, middle = 299} seatPreference

引用它

seatPreference bobSeatPreference = aisle;
